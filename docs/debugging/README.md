# docs/debugging/

## Purpose
A running log of real bugs you hit and how you fixed them — your own
personal troubleshooting reference, built from this exact project
instead of generic internet advice.

## Suggested format per entry

    ## Bug: <short description>
    **Symptom:** what you observed
    **Root cause:** what was actually wrong
    **Fix:** what you changed
    **Lesson:** what you'd check first next time

## Why bother with this
Two reasons: (1) you WILL hit the same class of bug again later in
this project or a future one, and a quick personal reference beats
re-discovering the fix from scratch; (2) "tell me about a hard bug you
debugged" is another very common interview question, and a real log
beats a reconstructed memory.
