# docs/learning_path/

## Purpose
A personal log of what you learned, in what order, and what confused
you along the way. This is for YOU, not really for recruiters — though
a polished version of this could become a great "what I learned
building this" blog post or LinkedIn write-up later (see the original
project plan's suggested next steps).

## Suggested structure
Create one entry per work session or per major milestone, e.g.:

    ## 2026-06-22 — Got ChromaDB ingestion working
    - What I built: ...
    - What confused me: ...
    - What I'd do differently: ...

## TODO
Keep this updated honestly as you go — the "what confused me" entries
are often the most valuable ones to look back on, and also make great
raw material for answering "what was the hardest part of this
project?" in an interview.
